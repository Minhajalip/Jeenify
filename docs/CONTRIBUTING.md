# Contributing Guidelines

## Repository Structure

frontend/ -- UI files backend/ -- Java backend database/ -- SQL scripts
docs/ -- Documentation tests/ -- Testing files

## Branch Strategy

main -- stable version develop -- integration frontend -- UI work
backend -- backend work database -- database work docs -- documentation
testing -- QA

## Workflow

1.  Pull latest code
2.  Work on feature
3.  Commit changes
4.  Push to branch
5.  Create Pull Request

## Commit Messages

Good examples: - Added login page - Created student table - Fixed
attendance bug

Avoid: - update - changes
